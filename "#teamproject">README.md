# Team Project – Software Engineering

## 📘 Project Overview
This repository contains the team project for Software Engineering.  
It demonstrates how to use GitHub for version control and collaboration.

## 🧩 Workflow
- Each team member works on a separate branch for their module.
- Changes are committed and pushed to GitHub.
- Pull requests are created for merging after code review.
- Merge conflicts are resolved manually when necessary.

## 🔀 Branching Strategy
- `main` – Stable production branch.
- `feature-login` – Login module development.
- `feature-database` – Database integration.
- `feature-ui` – User interface design.

## 🧠 Version Control Steps
1. Clone the repository.  
2. Create a new branch for your module.  
3. Make changes and commit them.  
4. Push your branch to GitHub.  
5. Create a pull request for review and merging.  
6. Resolve any conflicts before final merge.
